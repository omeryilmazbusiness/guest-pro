PHASE 4 — CONVERSATION MEMORY STRATEGY (NO MEMORY BLOAT)
==================================================

Implement a production-ready conversation memory strategy.

GOAL:
Do not keep unlimited full chat memory.
Keep only what is useful.

REQUIREMENTS:
- Store only the recent conversation window in full detail
- After a certain threshold, older messages should no longer stay in full memory context
- Instead, generate and store a compact rolling summary of the conversation topic/context
- The AI should remember only the latest topic context as a concise summary
- This summary should be enough to preserve continuity without bloating memory
- The exact threshold should be chosen professionally based on current architecture, but keep it practical and efficient

EXPECTED BEHAVIOR:
- recent messages remain visible and usable
- older messages are summarized into a compact memory field
- AI continues conversation with recent context + summary
- system remains performant and token-efficient
- no huge context history sent unnecessarily

ARCHITECTURE QUALITY:
- implement this server-side or in a robust backend-aware way where appropriate
- do not fake memory only in the UI
- keep it extendable for future long-term memory policies per hotel or per guest
