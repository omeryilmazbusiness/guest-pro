PROJECT CONTEXT:
Guest Pro is a hotel guest web app.
We now need an Active Tracking System that allows manager to define hotel geofence and allowed hotel network IPs, then track whether guests are:
- in hotel and on hotel wifi
- in hotel but not on hotel wifi
- outside hotel
- unknown

IMPORTANT PRODUCT INTERPRETATION:
This is a WEB APP solution.
Do NOT try to read SSID directly from browser.
Instead, the tracking decision must be made in backend using:
1. guest geolocation heartbeat
2. request source public IP matching against hotel allowed network IPs/ranges

PRIMARY GOALS:

2. Allow manager to define and edit:
   - hotel center latitude
   - hotel center longitude
   - tracking radius (default/use case: 100 meters)
   - allowed hotel public IPs or CIDR ranges
   - active/inactive tracking config
3. Compute guest tracking status in backend using a clean policy
4. Show guest tracking status in guest cards inside guest list using colored indicators
5. Make the whole tracking system easy to update when hotel location or wifi public IP changes
6. Keep everything SOLID and production-ready

   -> UNKNOWN
